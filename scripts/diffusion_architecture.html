Patchify — Latent to Token Sequence
A Conv2d with kernel = stride = patch_size converts the spatial latent into a flat token sequence
// DiT-XL/2 patchify Conv2d( in_channels = 4, // VAE latent channels out_channels = 1152, // hidden_size (16 heads × 72 dim) kernel_size = 2, // patch_size stride = 2 // non-overlapping ) // Data flow: input: (1, 4, 32, 32) ↓ Conv2d(k=2, s=2) output: (1, 1152, 16, 16) ↓ flatten + transpose (1, 256, 1152) // batch N hidden // ↑ // N = (32/2)×(32/2) = 256 patches
Why patch instead of per-pixel? A 32×32 latent has 1024 pixels — far too many tokens for self-attention (O(n²) cost). Patch size 2 reduces this to 256 tokens, which is manageable. Each patch covers a 2×2 region of the latent (which maps to ~16×16 pixels in the image via the VAE).
+ Positional embeddings: After patchifying, learnable position embeddings (256 × 1152) are added element-wise. These encode the 2D grid position of each patch so the transformer knows spatial arrangement.

What is adaLN-Zero?

Adaptive LayerNorm replaces the fixed γ, β parameters of standard LayerNorm with values predicted from the conditioning vector c. This lets the conditioning modulate how the normalized activations are scaled and shifted — different timesteps/classes produce different normalization parameters.

The "Zero" part: the gating parameters α₁, α₂ are initialized to zero. At training start, each block's contribution is zeroed out, making the entire network an identity function. This dramatically improves training stability for deep networks (DiT-XL has 28 blocks).

// Full adaLN-Zero equation per block: h = x + α₁ ⊙ Attn( γ₁ ⊙ LN(x) + β₁ ) out = h + α₂ ⊙ FFN( γ₂ ⊙ LN(h) + β₂ ) // At init: α₁ = α₂ = 0 // → h = x + 0 = x, out = x + 0 = x // → entire block is identity!

Dual Text Encoding — CLIP-L + T5-XXL
Two text encoders serve different purposes: global conditioning and token-level attention

CLIP-L (OpenAI)

Prompt → Tokenizer (77 max tokens) → CLIP text encoder → pooler_output: (768,) // single vector summarizing entire prompt

Role: Provides a global text summary that gets added to the timestep embedding. Tells the model "what kind of image" at every block through the adaLN conditioning vector.

T5-XXL (Google)

Prompt → Tokenizer (512 max tokens) → T5 encoder → sequence embeddings: (512, 4096) // one vector per token

Role: Provides a rich sequence of token embeddings that enter the transformer as "text tokens" in joint or cross attention. Lets the DiT attend to individual words — "golden retriever," "sunflowers," "oil painting."

Why two encoders? CLIP is trained on image-text pairs (knows visual semantics), while T5-XXL is a massive language model (understands nuanced text). Together they provide both visual alignment and deep language understanding.
Noise Initialization
Generation starts from pure Gaussian noise in the latent space
// FLUX.1 uses a 16-channel latent space (larger than Stable Diffusion's 4) target_size = 1024 × 1024 // desired image resolution vae_scale = 8 // VAE downsampling factor latent_size = 128 × 128 // 1024 / 8 latent_ch = 16 // FLUX latent channels z₁ = randn(1, 16, 128, 128) // pure noise at t=1 // mean ≈ 0, std ≈ 1 (standard Gaussian)
Why latent space? Working in pixel space would require denoising a 1024×1024×3 = 3.1M-dimensional signal. The VAE compresses this to 128×128×16 = 262K dimensions — a 12× reduction. Denoising is much faster and the latent captures semantic content, not pixel-level noise.
16 channels vs 4: FLUX uses a larger latent (16ch) compared to Stable Diffusion's 4ch. More channels means more information per spatial position, which helps capture finer details. The trade-off is more FLOPs per denoising step.

Also, flow matching creates straighter ODE trajectories than DDPM, requiring fewer Euler steps. No classifier-free guidance (CFG) is needed — it's been distilled into the model weights.

Dual-Stream Block (MMDiT-style)
Image and text maintain separate representations but share a single joint attention operation
// Dual-stream block forward pass // Image stream (3072-d per token) Qx = img_norm(x) @ W_Qx // separate QKV weights for image Kx = img_norm(x) @ W_Kx Vx = img_norm(x) @ W_Vx // Text stream (3072-d per token) Qy = txt_norm(y) @ W_Qy // separate QKV weights for text Ky = txt_norm(y) @ W_Ky Vy = txt_norm(y) @ W_Vy // Joint attention — concatenate before attention Q = [Qx ; Qy] // (4096+512, 3072) K = [Kx ; Ky] V = [Vx ; Vy] Attn = softmax(QKᵀ / √128) @ V // Split back into streams attn_img, attn_txt = split(Attn, [4096, 512]) // Separate FFNs per modality x = x + attn_img + img_FFN(img_norm2(x + attn_img)) y = y + attn_txt + txt_FFN(txt_norm2(y + attn_txt))

Why "Joint" Attention?

Instead of running separate self-attention for image and cross-attention for text, all tokens (image + text) are concatenated into one sequence before computing attention. This means every token attends to every other token in a single operation — more expressive than separate self-attention + cross-attention.

Why Separate FFNs?

Although attention is joint, the FFN is separate per modality. Image patches and text tokens have very different statistical properties — separate FFNs let each stream learn modality-specific transformations while still sharing information through joint attention.

Single-Stream Block
After dual-stream processing, all tokens are concatenated and processed with shared weights

Dual-Stream (×19)

  • • Separate QKV for img and txt
  • • Joint attention (concatenated Q/K/V)
  • • Separate FFN per modality
  • • ~2× params per block
  • • Maintains distinct modality streams

Single-Stream (×38)

  • • Single unified QKV for all tokens
  • • Standard self-attention
  • • Single shared FFN
  • • ~1× params per block (more efficient)
  • • All tokens treated identically
Design rationale: The dual-stream blocks (early layers) learn modality-specific transformations and align image and text representations. By the time tokens reach the single-stream blocks (later layers), the two modalities are already well-aligned and benefit from unified processing. This hybrid design balances expressiveness with parameter efficiency.
2D RoPE — Rotary Position Embeddings for Images
FLUX extends 1D RoPE (from LLMs) to 2D spatial grids by splitting the head dimension

Standard RoPE (1D — LLMs)

Token at position i: rotate all head dims by angle i·θ Encodes: relative distance between tokens in a 1D sequence

2D RoPE (FLUX — images)

Token at grid position (row, col): first half of head dims → rotate by row·θ second half of head dims → rotate by col·θ Encodes: relative 2D distance between patches on the spatial grid
// Example: 64×64 patch grid, head_dim=128 Token at (3, 5): dims [0:64] → rotated by 3·θ // encodes row=3 dims [64:128] → rotated by 5·θ // encodes col=5 Token at (3, 6): dims [0:64] → rotated by 3·θ // same row! dims [64:128] → rotated by 6·θ // 1 col apart // Attention between them naturally captures: // same row, 1 column apart

Advantages over Learned Position Embeddings

Resolution flexibility: RoPE works at any resolution without interpolation. Learned position embeds are fixed to the training resolution.

Relative awareness: RoPE encodes relative distances, not absolute positions. Two patches 3 apart always have the same relative encoding regardless of where they are in the image.

No extra parameters: RoPE is parameter-free — the rotation angles are computed from position indices and fixed frequency bands.

Flow Matching — The Math Behind Denoising
A simpler, more efficient alternative to the DDPM noise schedule
// Flow matching defines a straight path between noise and data // Forward process (creating noisy data): x_t = (1 - t) · x₀ + t · ε // linear interpolation // t=0: clean, t=1: noise // Velocity field (what the model predicts): v = dx_t/dt = ε - x₀ // direction from data to noise // Training objective: Loss = || f_θ(x_t, t) - v ||² // predict the velocity // Sampling (reverse — noise to data): x_{t-dt} = x_t - v · dt // Euler step (go backwards) // = x_t + v · (-dt)

Why Flow Matching over DDPM?

Straighter trajectories: The path from noise to data is approximately linear, so Euler's method with few steps gives a good approximation. DDPM's curved trajectory needs more steps for accuracy.

Simpler math: No noise schedule α_t, β_t to tune. Just a linear interpolation between clean data and noise.

Velocity vs noise: The model predicts a velocity (direction + magnitude) instead of just noise ε. This is more informative at low noise levels.

Euler Solver — Step-by-Step
The simplest ODE solver, following the predicted velocity field
// FLUX.1-schnell: 4 steps, t goes from 1.0 → 0.0 // Scheduler gives timesteps: [1.0, 0.75, 0.50, 0.25] Step 1: t=1.00, dt=-0.25 v₁ = DiT(z_1.00, t=1.00, text_cond) // model sees pure noise z_0.75 = z_1.00 + v₁ · (-0.25) // large step, rough structure Step 2: t=0.75, dt=-0.25 v₂ = DiT(z_0.75, t=0.75, text_cond) // model sees vague structure z_0.50 = z_0.75 + v₂ · (-0.25) // composition solidifies Step 3: t=0.50, dt=-0.25 v₃ = DiT(z_0.50, t=0.50, text_cond) // model refines mid-level z_0.25 = z_0.50 + v₃ · (-0.25) // details begin forming Step 4: t=0.25, dt=-0.25 v₄ = DiT(z_0.25, t=0.25, text_cond) // model adds fine details z_0.00 = z_0.25 + v₄ · (-0.25) // final clean latent!
Each step uses the SAME model with different timestep t. The model is trained to handle any noise level. Early steps (high t) make large structural changes. Later steps (low t) refine fine details. The timestep embedding tells the model what kind of corrections are needed at each stage.

What the Model "Sees" at Each Noise Level

t ≈ 1.0 (high noise)

Nearly pure noise. The model can only determine the coarsest properties: overall color distribution, rough spatial layout (sky above, ground below), approximate object positions.

t ≈ 0.75

Large-scale structure visible. The model resolves major compositional elements: where the subject is, background vs foreground, dominant colors and masses.

t ≈ 0.50

Medium-level features form. Object shapes are clear, boundaries sharpen, the overall scene is recognizable. Model refines proportions and spatial relationships.

t ≈ 0.25

Fine details emerge. Textures become visible — fur, leaves, reflections. Edges crisp up, subtle color gradients form. The image is nearly complete.

t ≈ 0.0 (clean)

Final refinement. The finest details are resolved — individual hairs, specular highlights, text readability. Ready for VAE decoding to pixels.

Coarse-to-fine generation: This hierarchical structure is not designed — it's an emergent property of training on all noise levels. The model naturally learns to make large-scale decisions first (when signal is weak) and fine-scale decisions last (when signal is strong). This mirrors how artists work: sketch → composition → details.
