The Core Idea: What Makes Them "Semantic"?

All three use a ViT backbone (the same architecture you already know). The magic is in what they train against. A ViT trained on ImageNet classification learns "this is a cat". A semantic encoder learns much richer representations because of its training signal:

CLIP learns features aligned with natural language. Its patch tokens "understand" arbitrary text concepts because they were trained to match 400M image-text descriptions.
SigLIP does the same as CLIP but with a better loss function. The sigmoid loss removes batch-size dependency, enabling more efficient training at scale.
DINOv2 learns features that capture visual structure — edges, textures, objects, spatial layout — without any language. Its features excel at dense spatial tasks.

CLIP — Contrastive Language-Image Pretraining DEEP DIVE

How CLIP trains two encoders to put images and text in the same embedding space

Dual Encoder Architecture
Both encoders are independent. They share no weights. The image encoder is a ViT. The text encoder is a separate GPT-like transformer. Both produce a single embedding vector (from [CLS] and [EOS] respectively), then a linear projection maps them into a shared 768-d space where cosine similarity is computed.
The Contrastive Loss — What Makes CLIP Learn
Batch of 8: similarity matrix I @ TT

Given a batch of N image-text pairs, CLIP computes cosine similarity between every image and every text — producing an N×N matrix.

# For batch of N image-text pairs: I = image_encoder(images) # (N, 768) T = text_encoder(texts) # (N, 768) # Normalize to unit vectors I = I / I.norm(dim=-1, keepdim=True) T = T / T.norm(dim=-1, keepdim=True) # Cosine similarity matrix logits = I @ T.T * exp(temperature) # (N, N) # The diagonal = matching pairs (should be HIGH) # Off-diagonal = non-matching (should be LOW) labels = arange(N) # [0, 1, 2, ..., N-1] loss_i2t = cross_entropy(logits, labels) loss_t2i = cross_entropy(logits.T, labels) loss = (loss_i2t + loss_t2i) / 2
Image → Text direction: For each image (row), the correct text should have the highest similarity. Softmax cross-entropy treats each row as a classification problem: "which of the N texts matches this image?"
Text → Image direction: Symmetrically, for each text (column), the correct image should score highest. Both directions are averaged. This ensures the alignment is bidirectional.
Why large batches matter for CLIP: The softmax normalizes across the entire batch. With N=32,768 (CLIP's actual batch size), each image has 32,767 negative texts to distinguish from. More negatives = harder task = better representations. This is also CLIP's weakness — it needs massive batches.
What CLIP Gives to VLMs
CLIP was never designed for VLMs! It was built for image-text matching using [CLS]. VLMs repurpose the intermediate patch tokens, which happen to contain rich, spatially-grounded, semantically-meaningful features — a lucky byproduct of contrastive training with text.

SigLIP — The Better Loss IMPROVED CLIP

Same dual-encoder idea, but sigmoid loss removes the batch-size dependency

CLIP Loss vs SigLIP Loss
# SigLIP loss — beautifully simple logits = I @ T.T * temperature + bias # (N, N) labels = 2 * eye(N) - 1 # +1 on diagonal, -1 off-diagonal loss = -logsigmoid(labels * logits).mean() # For a matching pair: labels=+1, want sigmoid(logit)→1 # For non-matching: labels=-1, want sigmoid(logit)→0 # Each cell is an independent binary classification!

DINOv2 — Self-Supervised, No Text Needed SELF-SUPERVISED

Learns visual features through self-distillation: a student learns to match a teacher's view of augmented images

Self-Distillation Training
# DINOv2 training loop (simplified) for images in dataloader: views = augment(images) # 2 global + N local crops # Teacher sees only global views (no gradients) with no_grad(): t_out = teacher(views[:2]) t_out = center(softmax(t_out / t_temp)) # sharpen + center # Student sees ALL views (global + local) s_out = student(views) s_out = softmax(s_out / s_temp) # Cross-entropy: student should match teacher loss = cross_entropy(s_out, t_out) loss.backward() # Only student gets gradients optimizer.step() # Teacher = slow-moving average of student teacher.params = ema * teacher.params + (1-ema) * student.params
Why does this work? The teacher provides a stable target (via EMA), while the student learns from diverse augmented views. Since both see the same content with different augmentations, the student learns augmentation-invariant features — which turn out to be semantically meaningful.
Local + Global crops: The student sees small crops that the teacher never sees. To match the teacher's global-view output from a local crop, the student must learn to infer global context from local patches — building rich, compositional features.
EMA (Exponential Moving Average) — The Teacher Update
# After each training step: for t_param, s_param in zip(teacher.params, student.params): t_param = momentum * t_param + (1 - momentum) * s_param # momentum = 0.996 → 0.9999 (cosine schedule) # Teacher is a "smoothed" version of the student # It changes slowly → provides stable training signal # Without EMA → training collapses (all outputs become identical)
Collapse prevention: Self-supervised methods can "cheat" by outputting the same vector for every input. DINO prevents this with: (1) EMA teacher (stable target), (2) centering (subtract mean of teacher outputs), (3) sharpening (low temperature in teacher softmax). This is the key challenge in self-supervised learning.

How VLMs Use These Encoders

The semantic encoder is just the first piece — here's how it connects to the rest

DINO for VLMs — the projector problem: Unlike CLIP/SigLIP, DINOv2 features are not text-aligned. The projector must work harder to bridge the gap. Some approaches combine CLIP (for semantic alignment) + DINO (for spatial detail) by concatenating their patch features before projection.

The Shared Embedding Space (CLIP/SigLIP)

The key innovation: images and text live in the same vector space

Zero-shot classification: To classify an image, embed it and find the closest text embedding from a set of class labels. No task-specific training needed — just describe your classes in text!
Why this matters for VLMs: Because CLIP features already "understand" text concepts, a simple MLP projector can bridge them to the LLM. The features are pre-aligned with language, making the projector's job much easier than with a non-text-aligned encoder like DINO.
